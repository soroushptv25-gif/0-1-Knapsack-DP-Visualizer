Decision Tree for Brute Force: At each node, we decide to INCLUDE (left) or EXCLUDE (right) the current item. The tree has depth n, generating 2ⁿ leaf nodes. For n=4 shown above, that's 16 subsets. DP avoids this by reusing subproblem solutions — pseudo-polynomial O(nW).
CLO-2: Value/Weight Ratio Sort — Initial Greedy Intuition
A natural first approach: sort items by value/weight ratio (most "efficient" first). This works for the fractional knapsack but can fail for 0/1. After filling the DP table, we trace back using this sorted order to understand the optimal solution.
DP Traceback Search
After building the DP table, we trace back from K[n][W] to find which items were selected. At each cell K[i][w]: if K[i][w] ≠ K[i-1][w], item i was included — move to K[i-1][w-wᵢ]. Otherwise, skip item i and move to K[i-1][w].

📚 Algorithm Overview▼

The 0/1 Knapsack Problem asks: given n items each with weight wᵢ and value vᵢ, and a knapsack of capacity W, which items should you take to maximize total value without exceeding capacity? Each item is either fully taken (1) or not (0) — you cannot take fractions.

O(2ⁿ)
Brute Force
O(nW)
Dynamic Programming
Why DP works: This problem has optimal substructure — the optimal solution to K[i][w] depends only on K[i-1][w] and K[i-1][w-wᵢ]. It also has overlapping subproblems — the same (i,w) pairs would be recomputed many times in recursion.
Pseudo-polynomial: O(nW) is not truly polynomial because W can be exponentially large as a binary number. If W = 2ⁿ, DP is no better than brute force. That's why this is "pseudo-polynomial."
🔍 Worked Example▼

Items: {A: w=2,v=3}, {B: w=3,v=4}, {C: w=4,v=5}, {D: w=5,v=6}. Capacity W=5.

1
Initialize K[0][w] = 0 for all w (no items, no value). Row 0 is the base case.
2
Item A (w=2, v=3): For w<2, can't include → K[1][w]=K[0][w]=0. For w≥2: max(K[0][w], 3+K[0][w-2]) = max(0, 3) = 3.
3
Item B (w=3, v=4): At w=5: max(K[1][5], 4+K[1][2]) = max(3, 4+3) = 7. Items A+B fit! Value 7.
4
Traceback from K[4][5]=7: K[4][5]=K[3][5]? No → D excluded. K[3][5]=K[2][5]? No → C excluded. K[2][5]=K[1][5]? 7≠3 → B included, move to K[1][2]. K[1][2]=K[0][2]? 3≠0 → A included.
5
Selected: A (w=2,v=3) + B (w=3,v=4). Total weight=5=W. Total value=7. ✅ Optimal!
💡 Key Concepts▼
Optimal Substructure: The optimal solution to K[i][w] can be built from optimal solutions to smaller subproblems — either K[i-1][w] (skip item i) or K[i-1][w-wᵢ]+vᵢ (take item i).
Overlapping Subproblems: Without memoization, a recursive approach recomputes K[i][w] exponentially many times. DP fills each cell exactly once.
Traceback: The DP table stores values, not decisions. To find which items were chosen, we trace back from K[n][W], checking at each step whether the value came from including or excluding the item.
Common Mistake: Confusing 0/1 Knapsack with Fractional Knapsack. Greedy (ratio sort) works for fractional but NOT for 0/1. Always countercheck with the DP result.
Space Optimization: Since row i only depends on row i-1, you can reduce space to O(W) by using a 1D rolling array (iterate w in reverse).
📊 Pseudocode & Recurrence▼
function KnapsackDP(weights, values, W, n):
  for i in 0..n, w in 0..W: K[i][w] = 0
  for i = 1 to n:
    for w = 0 to W:
      if weights[i] > w:
        K[i][w] = K[i-1][w]
      else:
        K[i][w] = max(K[i-1][w], values[i] + K[i-1][w-weights[i]])
  return K[n][W]
