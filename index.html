Decision Tree for Brute Force: At each node, we decide to INCLUDE (left) or EXCLUDE (right) the current item. The tree has depth n, generating 2ⁿ leaf nodes. For n=4 shown above, that's 16 subsets. DP avoids this by reusing subproblem solutions — pseudo-polynomial O(nW).

  if wi > w:  K[i-1][w]                        // can't include item i
  else:   max( K[i-1][w], vi + K[i-1][w-wi] )   // exclude vs include
📚 Algorithm Overview▼

The 0/1 Knapsack Problem asks: given n items each with weight wᵢ and value vᵢ, and a knapsack of capacity W, which items should you take to maximize total value without exceeding capacity? Each item is either fully taken (1) or not (0).

O(2ⁿ)
Brute Force
O(nW)
Dynamic Programming
Why DP works: This problem has optimal substructure and overlapping subproblems — the two conditions needed for DP.
Pseudo-polynomial: O(nW) is not truly polynomial — if W = 2ⁿ, DP is no better than brute force.
🔍 Worked Example▼

Items: {A: w=2,v=3}, {B: w=3,v=4}, {C: w=4,v=5}, {D: w=5,v=6}. Capacity W=5.

1
Initialize K[0][w] = 0 for all w. Base case: 0 items = 0 value.
2
Item A (w=2, v=3): For w≥2: max(0, 3+K[0][w-2]) = 3.
3
Item B (w=3, v=4): At w=5: max(3, 4+3) = 7. A+B fit!
4
Traceback: K[4][5]→K[3][5]→K[2][5]≠K[1][5] → B included → K[1][2]≠K[0][2] → A included.
5
Selected: A (w=2,v=3) + B (w=3,v=4). Total weight=5. Total value=7. ✅
💡 Key Concepts▼
Optimal Substructure: The best answer for K[i][w] uses best answers to smaller subproblems.
Overlapping Subproblems: Without memoization, K[i][w] is recomputed exponentially many times.
Traceback: The DP table stores values, not decisions. Traceback recovers which items were chosen.
Common Mistake: Greedy ratio sort works for fractional knapsack, NOT for 0/1.
Space Optimization: Since row i only needs row i-1, space can be reduced to O(W) with a 1D rolling array.
📊 Pseudocode & Recurrence▼
function KnapsackDP(weights, values, W, n):
  for i = 1 to n:
    for w = 0 to W:
      if weights[i] > w:
        K[i][w] = K[i-1][w]
      else:
        K[i][w] = max(K[i-1][w], values[i] + K[i-1][w-weights[i]])
  return K[n][W]
